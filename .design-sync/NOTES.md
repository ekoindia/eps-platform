# design-sync NOTES — eko-eps-website

Repo-specific gotchas for `/design-sync`. Read before re-syncing.

## Setup facts

- **Shape = package, synth-from-src.** This repo is a Vite/shadcn website, NOT a published component library — there is no library `dist/` entry. The converter bundles the branded set from `src/` via a hand-authored entry: `.design-sync/entry.tsx` (passed as `--entry`). It re-exports exactly the `componentSrcMap` names + `DSProvider`.
- **Provider** = `.design-sync/provider.tsx` → `DSProvider` (HelmetProvider + MemoryRouter). Branded cards/layouts/sections use react-router `<Link>` and some set `<Helmet>`; without the provider their floor renders crash. Exported from the entry so `cfg.provider.component: "DSProvider"` resolves on `window.EkoDS`.
- **Package manager = npm.** Repo has conflicting lockfiles (`bun.lock`, `bun.lockb`, `package-lock.json`); `package.json` scripts use npm and `package-lock.json` is newest. App `node_modules/` is already populated — no reinstall needed for the sync.
- **Converter deps** live isolated in `.ds-sync/` (esbuild, ts-morph, @types/react, playwright). esbuild's native binary postinstall is blocked by the repo's allow-scripts setting — after `npm i`, run `node .ds-sync/node_modules/esbuild/install.js` once.

## Styling

- **cssEntry = `.design-sync/styles-src.css`** — a STABLE copy of the app's compiled Tailwind (`dist/assets/index-*.css`, hashed name). Regenerate before every build: `npm run build` (or `npx vite build`) then `cp dist/assets/index-*.css .design-sync/styles-src.css`. Do NOT point cssEntry at `src/index.css` (unprocessed `@import "tailwindcss"`) or at the hashed dist filename (changes every build).
- Tailwind **v4**, CSS-first (`@theme inline` in `src/index.css`). Brand tokens: `--color-eko-gold/-hover/-light`, `--color-eko-navy/-light`, `--color-eko-slate`, `--color-eko-success`.

## Two converter patches required (synth-from-src only)

1. **esbuild JSX automatic.** Source uses `react-jsx` (no `React` import). Added `"jsx": "react-jsx"` to root `tsconfig.json` `compilerOptions` so esbuild's tsconfig auto-discovery compiles automatic JSX (else "React is not defined"). This is a tracked repo edit — keep it.
2. **tsconfig-paths isFile guard.** `@/lib/utils` collides: both `src/lib/utils.ts` AND `src/lib/utils/` (dir) exist; the staged paths plugin matched the bare directory first ("is a directory"). Patched `.ds-sync/lib/bundle.mjs` tsconfigPathsPlugin to require `statSync(...).isFile()`. **`.ds-sync/` is gitignored and re-staged from the skill each run — this patch must be RE-APPLIED on every re-sync** (or fixed upstream).

## Scope

- **Branded layer only, floor cards only** (user choice). 41 components: brand composites (EkoLogo, Cards, Industry/Solution cards, ApiChip, CodeBlock, DropdownGrid, BreadcrumbNav, Picture), page layouts, branded `sections/*`, representative `pricing/*`. Excluded: 22 generic shadcn `ui/` primitives, website chrome (Header/Footer/CommandPalette/Zoho/TalkToSales/megapanels/LanguageSelector), infra (ErrorBoundary/ScrollToTop/AnimatedRoutes/transitions/FadeIn/DefaultMeta/NavLink), and `AiHint` (LLM-metadata, not a design component).
- **guidelinesGlob: []** — the repo's `docs/*.md` are internal plans (ui-ux-improvement-plan, welcome-landing-storyboards, ai-agent-platform-status…), not design guidance. Deliberately not synced.

## Re-sync risks (what can silently go stale)

- **styles-src.css staleness.** It's gitignored and regenerated by hand. If a re-sync forgets the `vite build` + `cp`, components render against an OLD compiled CSS (missing newly-used utilities). Always rebuild dist first.
- **The two patches above** must survive re-stage; #2 (isFile guard) is the fragile one — re-staging `.ds-sync/` drops it.
- **4 near-blank floor cards** (ApiChip, DropdownColumnHeader, SectionContainer, StatCard) — render but PNG <5KB. Honest floor cards but visually empty; best first candidates to author `.design-sync/previews/<Name>.tsx` on a later sync.
- **Data-heavy components floor-card by design** (sections/pricing/page-layouts need real props/data). Expected, not a failure. 21/41 show the typographic floor card.
- **Tailwind safelist limitation.** The bundled CSS only contains utilities the app actually uses. The design agent gets the component-carried classes + brand tokens, but arbitrary Tailwind utilities it invents won't be in `_ds_bundle.css` unless the design environment recompiles Tailwind. Conventions header names only verified classes.
- **Bundle is large (~2.8MB)** — narrow entry still pulls radix, embla, react-hook-form, react-router transitively. Acceptable; trim the entry if it matters.
